# 鸿蒙 (HarmonyOS NEXT) App 开发调试经验

> 通用鸿蒙开发工具链、调试命令、沙箱与日志操作。记录的是「非显而易见、踩过坑才知道」的经验，不是官方文档能查到的内容。

---

## 1. 环境与工具链

- **hdc**（设备控制，类似 adb）：`/Applications/DevEco-Studio.app/Contents/sdk/default/openharmony/toolchains/hdc`
- **OHOS SDK**：`OHOS_SDK_HOME=/Applications/DevEco-Studio.app/Contents/sdk/default/openharmony`；构建前 `OHOS_BASE_SDK_HOME` 也要指向同一路径。
- **hvigor / ohpm**（构建 & 包管理）：`/Applications/DevEco-Studio.app/Contents/tools/{hvigor,ohpm}/bin` 加进 PATH。
- **OHOS 版 Flutter**：`~/.local/ohos-flutter-3.35.7/bin/flutter`（版本可能因项目而异）。

### 构建 HAP（ArkTS/打包改动，native .so 已就位时）
```bash
export OHOS_SDK_HOME=/Applications/DevEco-Studio.app/Contents/sdk/default/openharmony
export OHOS_BASE_SDK_HOME=$OHOS_SDK_HOME
export PATH=/Applications/DevEco-Studio.app/Contents/tools/ohpm/bin:/Applications/DevEco-Studio.app/Contents/tools/hvigor/bin:$PATH
flutter build hap --debug --target-platform ohos-arm64 --no-pub
```
产物：`ohos/entry/build/default/outputs/default/entry-default-signed.hap`。
安装：`hdc -t <target> install -r <hap>`（`-r` 保留应用数据）。

## 2. hdc / uitest 真机调试命令速查

| 目的 | 命令 |
|---|---|
| 启动 Ability | `hdc shell aa start -b <bundle> -a EntryAbility` |
| 传 **boolean** 参数 | `aa start ... --pb key true`（**布尔用 `--pb`，字符串才用 `--ps`**，混用会静默失效） |
| 传 deep link URI | `aa start ... -U 'scheme://host?...'` |
| 强杀应用 | `aa force-stop <bundle>` |
| 模拟点击 | `uitest uiInput click <x> <y>` |
| 输入文本 | `uitest uiInput inputText <x> <y> "text"` |
| 导出 UI 布局（找控件坐标/文本） | `uitest dumpLayout -p /data/local/tmp/lay.json` |
| 截屏 | `snapshot_display -f /data/local/tmp/x.jpeg` |
| 实时日志 / 清日志 | `hilog -x`（dump） / `hilog -r`（清空） |
| 网络&DNS 状态 | `hidumper -s NetConnManager` |
| 查看 tun 收发字节 | `ifconfig vpn-tun`（RX/TX bytes 增长 = 真的在过流量） |
| 反向端口（设备→Mac） | `hdc rport tcp:<dev> tcp:<mac>` |
| 正向端口（Mac→设备） | `hdc fport tcp:<mac> tcp:<dev>` |

**坑：**
- 设备上 **没有 `curl`**；要测代理连通性得用端口转发到 Mac 上 curl，或用设备内应用产生流量。
- `aa start` 在**锁屏**时报 `10106102` 失败 —— 测试脚本要先保证亮屏解锁。
- `uitest dumpLayout` 出来的 `bounds` 是 `[x1,y1][x2,y2]`，点击用中心点。

## 3. 应用沙箱与日志（读不到沙箱是最大障碍）

- 应用数据沙箱：`/data/storage/el2/base/...`（应用自己视角）。
- **`hdc shell` 用户读不到这个沙箱路径**（非 root），也 **不能在里面 `mkdir`**（permission denied）。
- **绕过办法**：同一份文件在**宿主真实路径**下可读：
  `/data/app/el2/100/base/<bundle>/haps/entry/files/`
  用 `hdc file recv <真实路径> <本地>` 拉出来看。
- **日志落盘技巧**：库内进程没有可绑定的 external-controller 时，让它把诊断信息写到 `.../files/xxx.log`（world-readable），再从宿主真实路径 recv 出来 —— 这是排查库内行为唯一可靠的手段。
- 想改沙箱里已存在的 world-writable 文件：`hdc file send` 可以覆盖，但**不能新建目录**。

## 4. ArkTS / hvigor 构建基础

- ArkTS 是 **strict TS**：`arkts-no-any-unknown` 会直接把 `any`/`unknown` 报成编译错误。回调签名要显式给类型。
- **插件注册漂移**：`GeneratedPluginRegistrant.ets` 会被工具重新生成，容易把手改的自定义注册**覆盖掉**。要么保持最小生成态、要么用测试钉住源码模式。
- 基本分析/测试：`flutter analyze --no-fatal-infos`、`flutter test`。
