# Web3 USDC 支付

## 主航道

```text
买家支付 USDC
-> Base 链确认
-> 地址归集
-> KYT 筛查
-> 兑换法币
-> 多币种账本
-> payout 结算商户
```

首期采用单链单币：USDC on Base。平台计划为商户分配收款地址，统一归集到托管钱包，再按账本进行分账和结算。

## Checkout 体验

Checkout 需要展示：

- 精确支付金额。
- 收款地址或二维码。
- 报价有效期和倒计时。
- 等待链上确认状态。
- 支付成功或异常状态。

买家必须填写邮箱；付款地址自动记录为支付属性，默认作为退款地址。

## 必须新增的能力

- RPC 节点接入和冗余。
- 链上入账监听、确认数策略和异常识别。
- PaymentProcessor 的 Crypto 实现。
- USDC 收款配置和订单 txhash 展示。
- 托管钱包、归集、签名服务和法币兑换。
- KYT 地址筛查、四方对账和多币种账本。

## 与卡支付的关键差异

Crypto 支付通常不可逆，没有卡拒付机制；退款改为平台通过备付池主动出金。因此“确认了什么、是否风险通过、钱是否已归集、退款资金从哪里出”都必须形成可追溯记录。
