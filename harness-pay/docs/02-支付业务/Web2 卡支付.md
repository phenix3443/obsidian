# Web2 卡支付

## 业务流程

```text
商户入驻审核
-> 创建商品/价格
-> 买家进入 Checkout
-> Stripe 卡授权与扣款
-> 税务/发票
-> 权益发放
-> 账本记录
-> 商户 payout
```

## 特点

- 采用成熟的 Stripe 收单能力。
- 支持一次性支付、订阅、优惠码、免费试用、席位和用量计费等 Polar 基线能力。
- 具备拒付争议处理，但需要平台承担相应的资金和运营风险。
- 是首批真实商户上线的基础轨道。

## 上线依赖

- 确定可承责的法律主体。
- 完成 Stripe 收单账户申请和业务描述。
- 准备美欧银行账户。
- 验证 Stripe Tax、发票模板和 payout 链路。
- 完成 AUP 重写与商户 KYB。
