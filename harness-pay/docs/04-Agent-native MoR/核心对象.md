# Agent-native MoR 核心对象

| 对象 | 含义 | 不变量 |
| --- | --- | --- |
| Principal | 承担预算、税务和法律后果的人或企业 | 每笔交易都有明确主体 |
| Agent Identity | Agent Provider、实例、版本、密钥和撤销状态 | Agent 不是默认法律买方 |
| Mandate | 对商户、类别、金额、频率、地区和时间的可撤销授权 | 超出范围必须拒绝或 step-up |
| Quote | 带价格、有效期、条款、预算和支付轨道的报价 | 最终金额不能超过授权边界 |
| Task | 可执行、可计量、可交付和可验收的任务 | Task 不等于最终 Sale |
| Usage | token、工具、运行时、重试等原始使用事实 | 可去重、可重放、不可静默修改 |
| Evidence | 结果哈希、签名、交付和验收证明 | 结算可映射到证据 |
| Sale / Invoice | MoR 与买方之间的法律销售和凭证 | 税务和商业价格可追溯 |

## 三本账

- Usage/Evidence：记录实际发生了什么。
- Commercial：记录客户按什么价格购买和计费。
- Money：记录预留、扣款、退款、分账、准备金和 payout。

三本账通过 quote、task、usage 和 transaction ID 关联，但不能混成一张余额表。
