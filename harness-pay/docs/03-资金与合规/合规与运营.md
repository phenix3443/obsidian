# 合规与运营

## 商户准入

- 重写 AUP，使合规的 Crypto/Web3 商户可以入驻。
- 接入 KYB，并把审核状态接入商户 onboarding。
- 审核法律主体、经营范围、目标买家区域和结算信息。

## 交易风控

- 对 Crypto 入账地址执行 KYT，覆盖制裁、黑名单、混币器等风险。
- 按金额或风险阈值触发买家 KYC，具体阈值需法律和产品评审。
- 对少付、多付、错链、错币、过期和异常速度建立处理规则。

## 运营后台

- 备付池和收款地址归集监控。
- txhash 查询和“转账未到账”排查。
- 链上监听健康度和日终对账差异视图。
- 客服只读、运营退款、超级管理员动资金的角色分级。
- 高风险退款、payout 账户变更和动资金操作需要审批和审计日志。

## 责任边界

税务申报与代缴依赖实际法律主体和国家范围，不是单纯研发功能。平台不能默认对所有国家、所有商品和所有 Crypto 场景开放。
